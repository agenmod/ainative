# 安全策略

## 报告漏洞

请使用 **GitHub Security Advisories**（若本仓库已开启 *Private vulnerability reporting*）：

1. 打开仓库主页 → **Security** → **Report a vulnerability**（或 **Advisories** → **Report**）。
2. 描述影响范围、复现步骤、受影响版本；**不要**在公开 Issue 中贴可利用细节或密钥。

若仓库未开启该功能，可通过 GitHub 支持文档联系仓库维护者开启，或使用维护者在 Profile 中公开的联系方式（**勿**在 Issue 中发送密钥）。

## 非安全问题

配置用法、功能建议、文档笔误请使用普通 Issues。

## 敏感信息

切勿在 Issue、PR、讨论中粘贴：生产密钥、数据库连接串、JWT、API Token。详见 [docs/SECRET_POLICY.md](docs/SECRET_POLICY.md)。
